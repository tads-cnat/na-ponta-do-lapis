FLUXO DE VALIDAÇÃO:

1. Frontend (Modal):
   ├─ addEventListener('submit') no formulário
   ├─ Validação JavaScript local
   │  ├─ Nome obrigatório e >= 3 caracteres
   │  ├─ Saldo obrigatório e formato numérico
   │  └─ Tipo selecionado
   ├─ Se erro: display erros no modal (preventDefault())
   └─ Se OK: submit() do formulário

2. Backend (View):
   ├─ POST em /contas/add_conta/
   ├─ Validação básica (campos preenchidos)
   ├─ Chamada para ContaService.AddContaService()
   │  ├─ Criação de objeto ContaFinanceira
   │  ├─ Chamada a full_clean()
   │  │  ├─ Validação do modelo (clean method)
   │  │  ├─ Validação de tipos against TIPOS_CONTA
   │  │  └─ Mensagens de erro específicas
   │  └─ Se ValidationError: raise
   ├─ Captura de ValidationError
   │  ├─ Extração de message_dict
   │  └─ Criação de messages.error() para cada campo
   └─ redirect('contas:index') com mensagens

3. Frontend (Página):
   ├─ Redirecionamento para /contas/
   ├─ Renderização do template
   ├─ Loop {% for message in messages %}
   │  ├─ Se erro: alert alert-error (vermelho)
   │  ├─ Se sucesso: alert alert-success (verde)
   │  └─ Se info: alert alert-info (azul)
   └─ Mensagem exibida ao usuário
