# ISO/IEC 25010

A ISO/IEC 25010 é uma norma internacional que define um modelo de qualidade para o produto de software, organizando-o em 8 características principais, cada uma com subcaracterísticas específicas.

---

## 1. Adequação funcional

**Subcaracterísticas:**
- **Correção:** O software atende corretamente aos requisitos.
- **Completeness:** Todas as funções necessárias estão implementadas.
- **Adequação:** Funcionalidades adequadas ao contexto de uso.
- **Funcionalidade de segurança:** Proteção contra acessos indevidos nas funções.

**Como testar:**
- Testes funcionais detalhados por requisito (BDD ou casos de uso).
- Verificar implementação de todas as funcionalidades exigidas por stakeholders.
- Testar fluxos não previstos (ex.: navegação extranormal).
- Validar permissões de acesso por função (admin, usuário comum).

---

## 2. Eficiência de desempenho

**Subcaracterísticas:**
- **Tempo de resposta:** Tempo entre entrada e resposta.
- **Utilização de recursos:** Uso de CPU, memória, disco.
- **Capacidade:** Número de transações ou usuários simultâneos.

**Como testar:**
- Testes de carga e estresse para medir tempo de resposta sob alta demanda.
- Monitorar uso de recursos durante testes.
- Aumentar progressivamente usuários até o limite suportado.

---

## 3. Usabilidade

**Subcaracterísticas:**
- **Apreensibilidade:** Fácil de entender.
- **Operabilidade:** Fácil de usar.
- **Proteção do usuário:** Erro não grave para o usuário.
- **Conformidade:** Adequação às normas de usabilidade.

**Como testar:**
- Testes de usabilidade com usuários reais gravando tarefas (ex.: cadastro).
- Avaliar clareza de mensagens de erro e forma de correção.
- Verificar padroes de UX (breadcrumb, mensagens claras).

---

## 4. Confiabilidade

**Subcaracterísticas:**
- **Maturidade:** Poucos erros.
- **Disponibilidade:** Disponível quando necessário.
- **Tolerância a falhas:** Capacidade de se recuperar sem perda de dados.
- **Recuperabilidade:** Capacidade de restaurar o estado.

**Como testar:**
- Testes de carga/estresse para falhas críticas.
- Testar uptime em produção.
- Simular falhas de hardware/banco e recuperar.
- Testes de backup/restauração.

---

## 5. Segurança

**Subcaracterística:**
- **Confidencialidade:** Proteção de dados.
- **Integridade:** Dados não alterados indevidamente.
- **Conformidade:** Obediência a normas de segurança.

**Como testar:**
- Testes de vulnerabilidade (OWASP Top 10).
- Verificar criptografia de dados sensíveis.
- Testar permissões de acesso e autenticação.

---

## 6. Manutenibilidade

**Subcaracterísticas:**
- **Modularidade:** Estrutura em módulos reutilizáveis.
- **Analisabilidade:** Facilidade de encontrar defeitos.
- **Modificabilidade:** Facilidade de alterar o código.
- **Testabilidade:** Facilidade de testar o código.

**Como testar:**
- Revisões de código para modularidade.
- Testes automatizados para cobrir novas funcionalidades.
- Verificar documentação e entendibilidade.
